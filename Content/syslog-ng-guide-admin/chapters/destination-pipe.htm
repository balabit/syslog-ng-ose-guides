pipe : Sending messages to named pipes

The pipe() driver sends messages to a named pipe like /dev/xconsole .

The pipe driver has a single required parameter, specifying the filename of the pipe to open. The filename can include macros. For the list of available optional parameters, see .

Declaration:

pipe(filename);

Caution:

Starting with abbrev 3.0.2, pipes are created automatically. In earlier versions, you had to create the pipe using the mkfifo(1) command.

Example: Using the pipe() driver

destination d_pipe { pipe("/dev/xconsole"); };
