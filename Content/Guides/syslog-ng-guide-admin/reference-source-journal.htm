systemd-journal() source options

The systemd-journal() driver has the following options:

match-boot()
Type: yes, no
Default: no

Available in 4.1 and later versions.

Description: When this option is set to yes, only fetches journal messages that relate to the current boot, and to ignores messages generated in previous boots.

matches()
Type: String
Default:

Available in 4.1 and later versions.

Description: This option specifies one or more filters to apply on the journal fields, similarly how you can use journalctl.

Example:
matches(
    "_COMM" => "systemd"
    )
