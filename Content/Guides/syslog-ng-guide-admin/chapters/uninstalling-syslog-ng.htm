Uninstalling

If you need to uninstall for some reason, you have the following options:

  • If you have installed using the .run installer: Execute the uninstall.sh script located at /opt/syslog-ng/bin/uninstall.sh. The uninstall script will automatically restore the syslog daemon used before installing syslog-ng. To completely remove , including the configuration files, use the uninstall.sh --purge command.

  • If you have installed from a .deb package: Execute the dpkg -r syslog-ng-premium-edition command to remove syslog-ng, or the dpkg -P syslog-ng-premium-edition command to remove and the configuration files as well. Note that removing does not restore the syslog daemon used before syslog-ng.

  • If you have installed from an .rpm package: Execute the rpm -e syslog-ng-premium-edition command to remove . Note that removing does not restore the syslog daemon used before .

  • If you have compiled from source: Execute the sudo make uninstall command to remove . Note that removing does not restore the syslog daemon used before .

  • If you have installed from a .pkg package: Execute the pkgrm BBsyslng command to remove . Note that removing does not restore the syslog daemon used before syslog-ng.

    For automatic uninstall (answering y to all questions): Execute the yes | pkgrm BBsyslng command.

    The following files have to be deleted manually:

    • <syslog-ng path>/etc/syslog-ng.conf

    • <syslog-ng path>/var/syslog-ng.persist

    • <syslog-ng path>/var/syslog-ng-00000.qf

    • anything else under the <syslog-ng path>/var directory
